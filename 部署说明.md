## 前置准备

### 1. 本地环境

- Node.js 16+ 
- npm
- SSH客户端

### 2. 云服务器

- Ubuntu 20.04+ / CentOS 7+ / Debian 10+
- 至少 1GB 内存
- 稳定的网络连接
- SSH访问权限

### 3. API密钥

- Kimi API Key（从 https://platform.moonshot.cn/ 获取）

## 本地开发

### 1. 配置环境变量

```bash
# 复制配置文件
cp .env.example .env

# 编辑配置文件，填入API Key
nano .env
```

在 `.env` 文件中设置：
```
KIMI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
```

### 2. 安装依赖

```bash

```

### 3. 本地运行

```bash
npm start
```

## 服务器部署

### 步骤 1: 服务器依赖安装

首先需要在云服务器上安装必要的依赖。

#### 方式 A: 通过SSH直接在服务器上执行

```bash
# 1. SSH登录到服务器
ssh user@your-server-ip

# 2. 下载setup脚本（如果你有git仓库）
wget https://your-repo/setup-server.sh
# 或者手动上传setup-server.sh文件

# 3. 添加执行权限
chmod +x setup-server.sh

# 4. 运行脚本
./setup-server.sh
```

#### 方式 B: 从本地上传并执行

```bash
# 1. 上传脚本到服务器
scp setup-server.sh user@your-server-ip:/tmp/

# 2. SSH登录并执行
ssh user@your-server-ip
chmod +x /tmp/setup-server.sh
/tmp/setup-server.sh
```

脚本会自动：
- ✅ 检测操作系统
- ✅ 安装 Node.js 20.x
- ✅ 安装 npm
- ✅ 安装基础工具（tar, curl, wget等）
- ✅ 创建部署目录 `/opt/wechaty-bot`
- ✅ 可选安装 PM2 进程管理器
- ✅ 可选创建 systemd 服务

### 步骤 2: 配置部署脚本

编辑本地的 `deploy-bot.sh` 文件，修改服务器配置：

```bash
nano deploy-bot.sh
```

修改以下配置：

```bash
SERVER_HOST="your-server-ip"           # 改为你的服务器IP
SERVER_USER="root"                     # 改为你的SSH用户名
SERVER_PATH="/opt/wechaty-bot"         # 服务器部署路径（通常不需要改）
SERVER_PORT="22"                       # SSH端口（如果不是22需要修改）
```

### 步骤 3: 执行部署

```bash
./deploy-bot.sh
```

部署脚本会自动：
1. ✅ 检查本地配置（.env文件）
2. ✅ 打包项目文件
3. ✅ 上传到服务器
4. ✅ 在服务器上安装npm依赖
5. ✅ 备份旧版本
6. ✅ 安装监控脚本
7. ✅ 询问是否立即启动

### 步骤 4: 启动机器人

部署完成后，可以选择以下方式启动：

#### 方式 A: 手动启动（简单测试）

```bash
ssh user@your-server-ip
cd /opt/wechaty-bot/current
node bot.js
```

#### 方式 B: 后台启动（使用nohup）

```bash
ssh user@your-server-ip
cd /opt/wechaty-bot/current
nohup node bot.js > logs/bot.log 2>&1 &
```

#### 方式 C: 使用PM2启动（推荐）

```bash
ssh user@your-server-ip
cd /opt/wechaty-bot/current
pm2 start bot.js --name wechaty-bot
pm2 save
pm2 startup  # 设置开机自启
```

#### 方式 D: 使用systemd启动

```bash
ssh user@your-server-ip
sudo systemctl start wechaty-bot
sudo systemctl enable wechaty-bot  # 开机自启
```

### 服务器监控

SSH登录后使用监控工具：

```bash
# 查看状态
monitor-bot status

# 持续监控日志（推荐）⭐
monitor-bot watch

# 实时监控面板（3秒刷新）
monitor-bot dashboard

# 实时监控面板（5秒刷新）
monitor-bot dashboard 5

# 查看最近日志
monitor-bot recent 100

# 查看错误日志
monitor-bot errors

# 搜索日志
monitor-bot search "关键词"

# 启动/停止/重启
monitor-bot start
monitor-bot stop
monitor-bot restart

# 查看帮助
monitor-bot help
```

### PM2 监控（如果使用PM2）

```bash
# 查看所有进程
pm2 list

# 查看详细状态
pm2 show wechaty-bot

# 查看日志
pm2 logs wechaty-bot

# 实时日志
pm2 logs wechaty-bot --lines 50

# 监控面板
pm2 monit

# 重启
pm2 restart wechaty-bot

# 停止
pm2 stop wechaty-bot

# 删除
pm2 delete wechaty-bot
```

### systemd 监控（如果使用systemd）

```bash
# 查看状态
sudo systemctl status wechaty-bot

# 查看日志
sudo journalctl -u wechaty-bot -f

# 最近100行日志
sudo journalctl -u wechaty-bot -n 100

# 重启
sudo systemctl restart wechaty-bot

# 停止
sudo systemctl stop wechaty-bot

# 启动
sudo systemctl start wechaty-bot
```

## 常见问题

### Q1: 部署脚本执行失败，提示 "请先配置SERVER_HOST"

**解决方案：**
编辑 `deploy-bot.sh`，修改顶部的服务器配置信息。

### Q2: SSH连接超时或连接被拒绝

**解决方案：**
1. 检查服务器IP是否正确
2. 检查SSH端口是否正确（默认22）
3. 检查服务器防火墙是否允许SSH连接
4. 使用 `ssh -v user@server-ip` 查看详细连接信息

### Q3: 服务器上找不到Node.js

**解决方案：**
先运行 `setup-server.sh` 安装依赖。

### Q4: 机器人无法登录微信

**可能原因：**
1. API Key配置错误 - 检查 `.env` 文件
2. 二维码已过期 - 重新启动机器人
3. 网络问题 - 检查服务器网络连接
4. 微信账号限制 - 尝试使用其他微信账号

### Q5: 如何查看机器人是否在运行？

```bash
# 方式1: 使用监控脚本
monitor-bot status

# 方式2: 查看进程
ps aux | grep bot.js

# 方式3: 使用PM2（如果安装了）
pm2 list

# 方式4: 使用systemd（如果配置了）
sudo systemctl status wechaty-bot
```

### Q6: 如何更新机器人代码？

```bash
# 在本地修改代码后，重新部署
./deploy-bot.sh

# 或者SSH到服务器手动更新
ssh user@server-ip
cd /opt/wechaty-bot/current
git pull  # 如果使用git
npm install  # 如果有新依赖
monitor-bot restart  # 重启机器人
```

### Q7: 日志文件太大怎么办？

```bash
# SSH到服务器
ssh user@server-ip

# 清理旧日志
cd /opt/wechaty-bot/current/logs
rm -f *.log.old
truncate -s 0 bot.log  # 清空当前日志

# 或者使用logrotate配置日志轮转
```

### Q8: 机器人频繁崩溃怎么办？

**排查步骤：**

1. 查看错误日志
   ```bash
   monitor-bot errors
   ```

2. 检查内存使用
   ```bash
   free -h
   ```

3. 查看进程资源占用
   ```bash
   top
   ```

4. 使用PM2自动重启（推荐）
   ```bash
   pm2 start bot.js --name wechaty-bot --max-restarts 10
   ```

### Q9: 如何备份和恢复？

**备份：**
```bash
ssh user@server-ip
cd /opt/wechaty-bot
tar -czf backup-$(date +%Y%m%d).tar.gz current/
```

**恢复：**
```bash
ssh user@server-ip
cd /opt/wechaty-bot
tar -xzf backup-20240101.tar.gz
```

### Q10: 如何配置多个机器人实例？

修改部署路径和端口，部署到不同目录：

```bash
# 编辑deploy-bot.sh
SERVER_PATH="/opt/wechaty-bot-1"  # 第一个实例
SERVER_PATH="/opt/wechaty-bot-2"  # 第二个实例
```

## 安全建议

1. ✅ 定期更新系统和Node.js
2. ✅ 使用SSH密钥登录，禁用密码登录
3. ✅ 配置防火墙，只开放必要端口
4. ✅ 不要将 `.env` 文件提交到Git仓库
5. ✅ 定期备份数据
6. ✅ 使用非root用户运行机器人
7. ✅ 监控日志中的异常行为

## 性能优化

1. **内存优化**
   ```bash
   # 限制Node.js内存使用
   node --max-old-space-size=512 bot.js
   ```

2. **使用PM2集群模式**（如果需要处理大量消息）
   ```bash
   pm2 start bot.js -i 2  # 启动2个实例
   ```

3. **配置日志轮转**，避免日志文件过大

4. **定期清理缓存**
   ```bash
   rm -rf /opt/wechaty-bot/current/.wechaty
   ```
