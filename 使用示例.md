# 使用示例

本文档提供实际的部署和使用示例，帮助你快速上手。

## 场景1: 首次部署到新服务器

假设你有一台全新的Ubuntu服务器，IP是 `192.168.1.100`，用户名是 `ubuntu`。

### 步骤1: 准备服务器

```bash
# 1. 从本地上传setup脚本到服务器
scp -P 22 setup-server.sh ubuntu@192.168.1.100:/tmp/

# 2. SSH登录服务器
ssh ubuntu@192.168.1.100

# 3. 运行setup脚本
chmod +x /tmp/setup-server.sh
sudo /tmp/setup-server.sh

# 按照提示操作：
# - 输入 y 安装 Node.js
# - 输入 y 安装 Git（可选）
# - 输入 y 安装 PM2（推荐）
# - 输入 y 创建 systemd 服务（可选）

# 4. 退出SSH
exit
```

### 步骤2: 配置部署脚本

在本地编辑 `deploy-bot.sh`：

```bash
# 使用你喜欢的编辑器
nano deploy-bot.sh
# 或
vim deploy-bot.sh
# 或
code deploy-bot.sh
```

修改配置：
```bash
SERVER_HOST="192.168.1.100"    # 改为你的服务器IP
SERVER_USER="ubuntu"           # 改为你的用户名
SERVER_PATH="/opt/wechaty-bot" # 保持不变
SERVER_PORT="22"               # 如果SSH端口不是22，修改这里
```

保存并退出。

### 步骤3: 执行部署

```bash
# 在本地项目目录执行
./deploy-bot.sh

# 看到类似输出：
# [INFO] 开始部署微信机器人到云服务器
# [INFO] 清理旧的打包文件...
# [INFO] 打包项目文件...
# [INFO] 上传到服务器 192.168.1.100...
# [INFO] 在服务器上部署...
# ...
# [INFO] 是否立即启动机器人？(y/n)

# 输入 y 立即启动
```

### 步骤4: 扫码登录

```bash
# SSH到服务器查看日志
ssh ubuntu@192.168.1.100
monitor-bot watch

# 你会看到二维码链接，复制到浏览器打开
# 使用微信扫码登录
```

登录成功后，机器人就开始工作了！

## 场景2: 更新机器人代码

假设你修改了 `bot.js` 中的系统提示词，想要更新到服务器。

### 方式A: 使用部署脚本（推荐）

```bash
# 1. 在本地修改代码
nano bot.js

# 2. 保存后直接部署
./deploy-bot.sh

# 脚本会自动：
# - 打包新代码
# - 上传到服务器
# - 备份旧版本
# - 部署新版本
# - 询问是否重启

# 3. 输入 y 重启机器人
```

### 方式B: 手动更新

```bash
# 1. SSH登录服务器
ssh ubuntu@192.168.1.100

# 2. 停止机器人
monitor-bot stop

# 3. 备份当前版本
cd /opt/wechaty-bot
cp -r current backup-manual

# 4. 更新代码（假设使用git）
cd current
git pull

# 5. 安装依赖（如果有新增）
npm install

# 6. 启动机器人
monitor-bot start
```

在本地开发时，你想实时查看日志和测试机器人。

```bash
# - 机器人运行状态

# - 最近15行日志
# - 每3秒自动刷新
```

## 场景4: 服务器日常监控

你的机器人已经在服务器上稳定运行，你想定期检查状态。

### 持续监控

```bash

monitor-bot dashboard

```

## 场景5: 使用PM2管理（推荐）

如果你在setup时安装了PM2，可以使用PM2管理机器人。

### 启动

```bash
cd /opt/wechaty-bot/current
pm2 start bot.js --name wechaty-bot

# 设置开机自启
pm2 save
pm2 startup
```

### 日常管理

```bash
# 查看状态
pm2 list

# 查看详细信息
pm2 show wechaty-bot

# 实时日志
pm2 logs wechaty-bot

# 监控面板
pm2 monit

# 重启
pm2 restart wechaty-bot

# 停止
pm2 stop wechaty-bot
```

### 自动重启配置

```bash
# 如果机器人崩溃，PM2会自动重启
# 配置最大重启次数
pm2 start bot.js --name wechaty-bot --max-restarts 10

# 如果内存超过500M自动重启
pm2 start bot.js --name wechaty-bot --max-memory-restart 500M
```

## 场景6: 紧急故障处理

机器人突然无响应，需要快速排查和修复。

### 步骤1: 快速诊断

```bash
# SSH登录
ssh ubuntu@192.168.1.100

# 检查进程是否运行
monitor-bot status

# 如果显示未运行，查看错误日志
monitor-bot errors
```

### 步骤2: 常见问题快速修复

#### 问题1: 进程卡死

```bash
# 强制停止并重启
pkill -9 -f "node.*bot.js"
monitor-bot start
```

#### 问题2: 内存不足

```bash
# 查看内存
free -h

# 清理日志
cd /opt/wechaty-bot/current/logs
rm -f *.log
truncate -s 0 bot.log

# 重启机器人
monitor-bot restart
```

#### 问题3: 端口被占用

```bash
# 查看端口占用（如果机器人使用了端口）
ss -tuln | grep 端口号

# 杀掉占用端口的进程
fuser -k 端口号/tcp

# 重启机器人
monitor-bot start
```

### 步骤3: 回滚到备份版本

如果新版本有问题：

```bash
cd /opt/wechaty-bot
monitor-bot stop
rm -rf current
mv backup current
monitor-bot start
```

## 场景7: 多机器人部署

你想在同一台服务器上运行多个机器人实例。

### 部署第一个机器人

```bash
# 编辑deploy-bot.sh
SERVER_PATH="/opt/wechaty-bot-1"

# 部署
./deploy-bot.sh
```

### 部署第二个机器人

```bash
# 创建第二个部署脚本
cp deploy-bot.sh deploy-bot-2.sh

# 编辑配置
nano deploy-bot-2.sh
# 修改：
SERVER_PATH="/opt/wechaty-bot-2"

# 创建第二个.env文件
cp .env .env.bot2
# 编辑不同的API Key

# 部署
./deploy-bot-2.sh
```

### 分别管理

```bash
# SSH到服务器

# 启动第一个
cd /opt/wechaty-bot-1/current
pm2 start bot.js --name wechaty-bot-1

# 启动第二个
cd /opt/wechaty-bot-2/current
pm2 start bot.js --name wechaty-bot-2

# 查看所有
pm2 list
```

## 场景8: 定时任务和自动化

设置定时任务，自动备份和监控。

### 自动备份

```bash
# SSH到服务器
ssh ubuntu@192.168.1.100

# 编辑crontab
crontab -e

# 添加以下内容：
# 每天凌晨2点备份
0 2 * * * tar -czf /opt/backups/wechaty-bot-$(date +\%Y\%m\%d).tar.gz /opt/wechaty-bot/current

# 删除30天前的备份
0 3 * * * find /opt/backups -name "wechaty-bot-*.tar.gz" -mtime +30 -delete
```

### 自动重启（如果崩溃）

```bash
# 使用PM2自动重启（推荐）
pm2 start bot.js --name wechaty-bot --max-restarts 10

# 或者使用systemd
sudo systemctl enable wechaty-bot

# 或者使用crontab（每5分钟检查一次）
*/5 * * * * pgrep -f "node.*bot.js" > /dev/null || cd /opt/wechaty-bot/current && node bot.js > logs/bot.log 2>&1 &
```

### 监控告警

```bash
# 每小时检查错误日志，如有错误发送通知
0 * * * * error_count=$(grep -c ERROR /opt/wechaty-bot/current/logs/bot.log); [ $error_count -gt 10 ] && echo "机器人错误数: $error_count" | mail -s "机器人告警" your@email.com
```

## 实用技巧

### 技巧1: 快速查看最新消息

```bash
# 只看最新的消息日志
monitor-bot watch | grep "收到消息"
```

### 技巧2: 统计消息数量

```bash
# 统计今天收到的消息数
cd /opt/wechaty-bot/current/logs
grep "收到消息" bot.log | grep "$(date +%Y-%m-%d)" | wc -l
```

### 技巧3: 导出日志

```bash
# 导出今天的日志
grep "$(date +%Y-%m-%d)" /opt/wechaty-bot/current/logs/bot.log > today.log

# 下载到本地
scp ubuntu@192.168.1.100:/opt/wechaty-bot/current/today.log ./
```

### 技巧4: 快速重新登录

如果微信掉线需要重新登录：

```bash
# 重启机器人
monitor-bot restart

# 立即查看登录二维码
monitor-bot watch | grep "qrcode"

# 复制链接到浏览器，扫码登录
```
