# 快速开始指南

## 第一步: 获取 Kimi API Key

1. 访问 Moonshot AI 开放平台: https://platform.moonshot.cn/
2. 注册并登录账号
3. 在控制台点击"API Keys"
4. 点击"创建新的 API Key"
5. 复制生成的 API Key (格式类似: sk-xxxxxxxxxxxxxxxxxxxxxxxx)

## 第二步: 配置项目

在项目根目录创建 `.env` 文件:

```bash
# 复制示例文件
cp .env.example .env

# 编辑 .env 文件
nano .env
# 或使用其他编辑器
```

在 `.env` 文件中填入你的 API Key:

```
KIMI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
```

保存并退出。

## 第三步: 启动机器人

```bash
npm start
```

## 第四步: 扫码登录

程序启动后,终端会显示类似这样的内容:

```
扫描二维码登录: 0
https://wechaty.js.org/qrcode/xxxxxxxxx
```

1. 复制链接并在浏览器中打开
2. 使用你想要作为机器人的微信账号扫描二维码
3. 在手机上确认登录

看到 "登录成功" 提示后,机器人就开始工作了!

## 使用示例

### 私聊测试

给机器人微信账号发送消息:

```
你: 你好
机器人: 你好!我是一个基于 Kimi 大模型的 AI 助手,很高兴为你服务...
```

### 群聊测试

1. 将机器人账号拉入一个测试群
2. 在群里 @ 机器人并发送消息:

```
你: @机器人昵称 讲个笑话
机器人: @你 好的!这里有一个笑话...
```

## 常见问题

### Q: 运行时报错 "请在 .env 文件中配置 KIMI_API_KEY"

A: 说明没有正确配置 API Key,请检查:
- `.env` 文件是否存在
- 文件中是否有 `KIMI_API_KEY=xxx` 这一行
- API Key 是否正确(不要有多余的空格或引号)

### Q: 机器人不回复消息

A: 可能的原因:
1. 检查终端是否有错误日志
2. 确认 API Key 是否有效
3. 私聊模式下,确保不是机器人自己发的消息
4. 群聊模式下,确保正确 @ 了机器人

### Q: API 调用失败

A: 检查:
- 网络连接是否正常
- API Key 是否有剩余额度
- 终端显示的具体错误信息

## 停止机器人

在终端按 `Ctrl + C` 即可停止程序。
